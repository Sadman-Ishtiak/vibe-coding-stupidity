# Development Tools and Documentation

This document lists the tools, libraries, and frameworks used in the development of InterNova, categorized by their role in the application lifecycle.

## 1. Frontend Development (Client)

| Tool | Purpose | Documentation |
| :--- | :--- | :--- |
| **React** | Component-based UI library | [Official Docs](https://react.dev/) |
| **Vite** | Next-generation frontend tooling (Build tool) | [Official Docs](https://vitejs.dev/) |
| **React Router** | Client-side routing and navigation | [Official Docs](https://reactrouter.com/) |
| **Bootstrap 5** | Responsive CSS framework | [Official Docs](https://getbootstrap.com/docs/5.0/getting-started/introduction/) |
| **Sass** | CSS preprocessor for advanced styling | [Official Docs](https://sass-lang.com/documentation/) |
| **Axios** | Promise-based HTTP client for API calls | [Official Docs](https://axios-http.com/docs/intro) |
| **Swiper.js** | Modern mobile touch slider | [Official Docs](https://swiperjs.com/get-started) |
| **Choices.js** | Lightweight, configurable select box library | [Official Docs](https://github.com/Choices-js/Choices) |
| **ESLint** | Pluggable linting utility for JavaScript | [Official Docs](https://eslint.org/) |

## 4. Internal Project Documentation

The following internal documentation files are available in the project root:

*   `README.md`: Project overview and setup instructions.
*   `API_ENDPOINTS.md`: Detailed list of all server routes and usage examples.
*   `SYSTEM_FLOW.md`: Technical breakdown of the Job and Application lifecycles.
*   `SECURITY_AUDIT_REPORT.md`: Analysis of security measures and vulnerability protections.
*   `LIMITATIONS_AND_FUTURE_SCOPE.md`: Roadmap for scaling and feature enhancements.
*   `audit files/`: A directory containing various implementation guides and audit reports.
